# Agent 愿景草稿 (Agent Vision)

## 维护原则

**只关注"agent 作为知行核心概念"的愿景、产品理念与架构方向**,不维护变更记录、状态进度、历史演化等无效信息。

- **范围**:本文讨论的是 **agent**(智能体本身作为知行的核心抽象与产品形态),**不是** sub-agent(子 agent 是 agent 概念之下的一种派生/组合形态,属下游话题)。本文先把上游的"我们要的 agent 是什么"想清楚,再谈任何派生设计。
- **写什么**:调研(业界与第一性原理)/ 产品理念(知行的 agent 要承载什么、不承载什么)/ 架构方向(为支撑该理念,抽象与边界应如何分层)/ 待对齐的开放问题。
- **不写**:修订记录 / 决策时间戳 / 状态进度标签 / 对历史讨论的回应 / 实现细节(schema、字段、文件路径、API 形态——那些留到 spec 阶段)。
- **更新方式**:原地改,不追加历史段;结论变了直接覆盖,不留旧版本注释。历史与演化留给 `git log`。
- **协作方式**:分批渐进对齐,一次只推进一个 Phase——助理列该 Phase 的待定问题并附倾向,**每条问题下必须留一行"用户回复:"占位**供用户就地作答(用户原话作为工作期对话痕迹留在本文,沉淀时不带入归档);用户回复后助理把共识提炼为"对齐结果"(聚焦方向、不展开实现),再进下一 Phase。

## 调研

> 维度锁定:产品定位 → 主/子 agent 模型强弱分配 → 集群/组合形态。下列为各产品**设计方向缩略**(非官话),不预设知行结论。

**Claude Code(编程 agent)**:定位生产级编码、追求产出质量,故主循环用最强模型(Opus)做推理/架构/综合,子 agent 按任务降配——实现交 Sonnet,检索/只读探索默认 Haiku(便宜约 15×)。核心理念:把"必须强模型"的认知任务与"弱模型够用"的机械任务分离,强脑在上、廉价手在下;二维控制面 = 模型档 × 思考力。

**OpenClaw(通用个人助手)**:定位广覆盖、成本敏感,主张"没有最佳模型",主 agent 用中档六边形模型(Sonnet 级,接近 Opus 能力但成本几分之一)。模型无关、可路由;典型反向配置 = 强 orchestrator + 廉价子 agent(DeepSeek)省 40–60% 成本;设能力下限(弱模型会选错工具/打转)。理念:成本/能力/速度三角权衡,交由用户调。

**Hermes(自托管个人助手,Nous)**:定位隐私、无供应商锁定、长期陪伴复利成长。模型完全无关,一句命令换模型;子 agent 可按配置自动路由到不同模型,会话/终端隔离;极端隐私可本地 Ollama 跑、API 成本归零。理念:用户拥有模型选择与数据主权,agent 越用越强,反 SaaS 锁定。

**Kimi(Agent Swarm / agent 集群,Moonshot)**:核心是自组织集群——经强化学习(PARL)训练的 orchestrator 把目标拆成可并行子任务,动态实例化最多 100–300 个领域子 agent(各带定制 prompt+工具),三层 = 编排/领域 agent/聚合。理念:靠"训练出来的编排智能"而非 prompt,以动态分工 + 大规模并行换速度与规模;子 agent 是临时冻结实例。

**Manus(通用自主 agent)**:定位全自主完成项目,非对话/流程器。形态 = "高管带专业团队":orchestrator 把复杂任务委派给规划/检索/codegen/校验等专职子 agent 并行协作。模型 = 强核心(Claude)+ 微调廉价模型(Qwen)分担子角色。理念:不堆单一巨模型,以专业化团队换更细腻高效。

> 来源:Kimi Agent Swarm(kimi.com/blog/agent-swarm、datacamp)、Claude Code model-config(code.claude.com/docs)、OpenClaw 模型选择(getopenclaw.ai、deepinfra)、Hermes(hermes-agent.nousresearch.com、turingpost)、Manus 架构(workos.com、manus.im/blog)。

## 产品理念

> 当前工作思路(**未定稿**,待按 Phase 对齐):知行是通用助手 + 覆盖工作场景。

知行 = 通用助手 + 内置工作场景级工作体。main 用中档模型、持"关于你"的 main 记忆,负责沟通、理解、路由与关系连续性。进入工作即在知行内开出一个工作场景作用域的工作体:power 模型 + 该工作场景专属记忆(像内置一个 claude code),一个工作场景一套,彼此隔离。记忆单向:main 可读各工作场景记忆以维系叙事,工作场景不可反写 main,杜绝工作场景细节稀释通用助手。工作场景是记忆单位,与作为权限安全区的 workspace 正交。每个工作场景稳定前缀天然 cache 友好。真需求锚点:价值在"知记得你、行替你干、无缝衔接"的连续性,而非工作场景级 claude code 启动器——后者即伪需求,守此分水岭。

> 关键澄清:这不是 sub-agent(委派、非交互、返结果),也不是把 main 换模型(毁 main 缓存);而是"换模式不换人"——同一场用户对话内进入工作模式,换更强的脑子但工具/工作区能力仍在、用户全程交互。

## 架构方向

> 当前工作思路(**未定稿**,待按 Phase 对齐)。

**统领原则:模型负责的地方做薄,不变量负责的地方做厚。**

模型只会越来越强。会过时的架构是把智能堆进脚手架(任务分类器、固定编排、能力补偿、为弱注意力做的复杂压缩)——模型变强后这些不是升级,是死重。经得起时间的架构反过来:

- **做薄**(让模型尽情变强,不拿脚手架罩它):干活本身、要不要换模式、怎么拆活,交给模型判断。模型越强自动越好。
- **做厚**(无论模型多强都必须成立的不变量):记忆边界、信任/权限边界、关系连续性、把对的上下文喂给模型。模型越自主,这些越吃重。

判定标尺:凡随模型变强而增值的,做厚;凡模型一强就成死重的,不做。

**架构五块:**

1. **两个记忆域 + 单向阀**。个人记忆(关于你)归 main,关系层、长寿。工作场景记忆(关于这摊活)一个工作场景一套,归该工作场景的工作会话。阀门单向:main 可**读**工作场景记忆以维系叙事;工作会话**永不能写**个人记忆;工作场景间互不串。单向性靠**架构构造**保证(工作会话物理上无写个人记忆的能力),不靠约定。
2. **"工作场景"是工作语义边界(记忆 + 工作前缀的归属),与 workspace(权限边界)各管各的——一个工作场景的工作目录可以在任何地方,跟 workspace 没关系**。workspace = 权限安全区;工作场景 = 一套工作场景记忆 + 一段稳定工作前缀(工作场景记忆 + 工作系统提示),一个工作场景一套,跨会话常驻热缓存,天生满足 cache 第一优先。main 维护"你有哪些工作场景"的登记,作为跨工作场景跨时间串叙事之根。**边界的形状由用户定义**——粒度、组织方式、生命周期均由用户决定,系统只提供接入 / 移除 / 列表 / 切换这套机制原语,不预设策略(机制与策略分离)。接入与移除是关系层动作,LLM 可提议但不可单方面执行,**必须用户拍板**;两个操作入口(通过 main 让 agent 操作 / 用户系统指令)共享同一组底层原语,仅暴露方式不同。当前能想到"接入 / 未接入"两态,不锁未来扩展。
3. **换模式是运行配置切换,不是委派、不是原地换模型**。普通模式 = main(中档、个人记忆、随时在、缓存热),处理生活+轻活+判断。工作模式 = 绑定具体工作场景的会话(power 模型、该工作场景记忆、全工具全工作区、**全程交互**),干完退回 main。不是 sub-agent(隔离/不交互/返结果——另留作研究员工具)。不在 main 自己那条线换模型(毁 main 缓存)。退出时一句纪要**追加 main 历史末尾**(尾部追加不毁前缀缓存),要长期记的写进**工作场景记忆**。**切换对用户明示但分通道**:server 通道不主动提醒(由调用客户端 UI 负责),cli 通道在切换点加分隔 + 一行弱提示告知,保持流不打断。
4. **换模式由模型判断,不是写死的分类器**。main 用自然判断提议、或用户直说;切换便宜可逆(先试→发现是真活→提议/切→随时可退)。**"先试"仅适用模糊场景**——信号明确(明确切 / 明确不切 / 关系层用户拍板)直接判断或交用户即可,不必"试";模糊场景下模型先在 main 里探线索(自身能力),探出再提议切,探不出来再问。决策交给模型 → 模型越强决策越准 → 架构质量随模型进步自动上涨。这是"时间会证明对"的核心来源,与静态配置(永远是固定用户负担)相反。
5. **连续性层 = 真正的产品**。main 持有:你是谁、有哪些工作场景、关系线索、对各工作场景记忆的只读权。于是 main 能不被问就串起跨工作场景跨时间的叙事——价值正是这条贯穿生活与工作、又把工作妥善隔开的关系。**只读权的形态是按需检索**:用户提及触发 → 模型识别 → 调一个统一接口查工作场景记忆(参数支持场景名 / 关键词 / 时间 / 状态等),返回片段或摘要;不预加载 / 不推送 / 不常驻 main,避免污染 main 注意力窗口。哪个场景相关由模型自然判断,不在机制层预判。

**必须焊死的不变量(真风险点)**:回 main 时既要叙事跟上、又不能污染个人记忆/撑爆 main 上下文。纪律:main 对工作场景记忆**按需拉取(读)非推送**;工作模式只写工作场景记忆 + 往 main 末尾追一句纪要;个人记忆只接收 main 自判"关于这个人"的条目,永不从工作模式自动灌入。此单向阀 by construction,不靠约定。
