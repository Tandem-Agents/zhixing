# /skill-new 进入体验 · 重设计草案（已废弃）

> 临时文档,对齐"一个不懂设计的真实用户进来该是什么体验"。经五视角(PM/UX/UI/零认知
> 用户/乔布斯式)评审后**收敛**:删掉过度的"概括确认闸门",把四屏向唯一的草稿屏塌缩,
> 并修掉四处硬伤。定稿后并入 [skill-authoring.md](../specifications/skill-authoring.md)。
> 效果图为 ASCII 示意,落在知行既有视觉语言(chrome 圆角框 / ▎章节锚 / ◈▣■◆ 印鉴流转 /
> 说明亮+按键暗 hint / dim 弱化 / ░ 骨架占位)。

## 一、为什么重设计

站在**第一次进来、不懂任何设计**的用户视角,现状(进屏 autoDraft 即转)会让他连环懵:
怎么就起草了 / 用我哪些话 / 出来是啥、我能改吗。根子上这一屏没回答三件事——**这一步
干嘛、它拿什么起草、用户怎么介入**。

但治"懵"不能滑向另一个反模式:**用多一屏、多一句解释、多一次确认去治"不放心"**。
真正的解是让产品本身不需要解释。所以这版的纲领是一句话:

> **草稿屏即一切。** 用户的动作本质只有两拍——"审一眼草稿" + "对了存、不对说句话改"。
> 开场、起草中、保存都不是独立的"产品页面",而是**同一个草稿屏的几个状态**,用户几乎
> 不该感觉到"屏在切换"。

三条贯穿原则不变:**知情**(每一步都看得见在发生什么、用了什么)、**控制**(关键节点有
介入点)、**预期**(始终知道下一步、产出长什么样)——但实现它们靠"把信息长进草稿屏",
而非"多加一屏来说明"。

## 二、核心反转(相对上一版)

| 上一版                                                         | 收敛版                                                                    | 为什么                                                                                                                    |
| -------------------------------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 有对话先**AI 概括 → 回车确认**(独立闸门 + 一次额外 LLM) | **删掉**。进屏直接起草,"收的是哪件事"作为草稿屏顶部 `收自:X` 呈现 | 对多数猜对的用户,确认是纯过路点头、还赔一次 LLM;把 0 决策硬塞成 1 决策。确认/知情/纠正合并进草稿屏一个动作,猜错就说句话改 |
| "起草中"是**独立等待屏** + 安抚文案                      | 起草中 = 草稿区**灰色骨架 → 字段就地显影**                         | 用户始终盯同一个屏看草稿成形,不切到另一屏干等;"几秒就好"这种安抚话本身暴露"等太久"是真问题                                |
| 四屏 + 三分支 + 一次额外 LLM                                   | **一句话 → 草稿就地显影 → 就地策展 → 存**                        | 决策点压到只剩"存,还是说句话改"                                                                                           |

## 三、流程总览

```
  /skill-new
      │
      ├─ 带首句意图 ───┐
      ├─ 有对话(没意图) ┼─→ 直接起草（草稿屏顶部显示「收自:X」，零额外 LLM）
      │                │         │
      └─ 冷启动无对话 ──┘         │  先停在草稿屏「等意图」态，你说一句再起草
                                  ↓
                  ┌───────────────────────────────────┐
                  │  草稿屏（一个屏，四个状态）          │
                  │  ① 等意图  ② 显影  ③ 策展  ④ 落地   │ ←─ 说句话改 ─┐
                  └───────────────────────────────────┘              │
                       │              │              │               │
                    Ctrl+S         说句话         Esc/Ctrl+E ─────────┘
                       ↓            (回②显影)        ↓
                    ④ 存好了                     放弃(二次确认) / 自己写
```

主路就一个屏。任何时候 Esc 放弃(有草稿则二次确认),Ctrl+E 转自己的编辑器。

## 四、逐状态:描述 + 效果图

### 状态 ① 等意图 —— 只在"没东西可起草"时出现(冷启动 / 无对话无首句)

这是草稿屏的**未填充态**:上方是空的草稿位,下方输入框让你说想收什么。带了首句意图、
或有对话可提炼时,**跳过本态、直接进 ② 显影**(不再让用户白点一次回车)。

```
╭ 新建技能 ───────────────────────────────────────────────────╮
│                                                             │
│   把你的一套做法，收成一个以后能一句话唤起的技能。           │
│                                                             │
╰─────────────────────────────────────────────────────────────╯

  技能 = 教我一次「这类事这么做」，往后照做、不用重讲。

 ▎想收个什么技能？说一句，我来写成草稿。
 ╭─────────────────────────────────────────────────────────────╮
 │ 比如：我审查 PR 的固定清单                                    │
 ╰─────────────────────────────────────────────────────────────╯
  自己写 Ctrl+E                                       起草 Enter
```

> · "技能 = …"那句**只在技能库为空时**出现(新手认知),库非空(已懂)去掉。
> · 给一条手写直通车 `自己写 Ctrl+E`:有明确想法、嫌跟 AI 来回慢的老手,直接进编辑器,
> 跳过 AI 起草整条链(默认仍是 AI 起草,但不禁止手写)。

### 状态 ② 显影 —— 起草不是干等,是草稿就地长出来

不切屏。草稿区先摆灰色骨架占位(`░`),印鉴 `◈▣■◆` 转(250ms/帧,复用 status-bar ticker),
字段一就位就**就地显影**成真内容。顶部 `收自:X` 让你立刻看到它在收哪件事——这就是上一版
"概括确认"想给的知情,现在不花一屏、不花一次 LLM 就给了。

```
╭ 新建技能 · 草稿 ────────────────────────────────────────────╮
│                                                             │
│   收自：配置生产环境的部署流程                                │
│                                                             │
╰─────────────────────────────────────────────────────────────╯

  ◈ 正在写…

  名称       ░░░░░░░░░
  什么时候用  ░░░░░░░░░░░░░░

  ── 正文 ──────────────────────────────────────────────────

  ░░░░░░░░░░░░░░░░░░░░░░
  ░░░░░░░░░░░░░░░
```

> · 改写回流(已有草稿、说句话改)走同一态,但文案换成增量语气:`◈ 正在按你说的改…`
>
> + 小灰字 `在现在这版上改，原来的会留着`,让用户信"改是增量、安全",而非推翻重写。
>   · "停下"全程只暴露一个键:与草稿屏一致用 **Esc**(Ctrl+C 底层仍兜底,但不主推,免得
>   它"杀程序"的强语义吓人)。首次起草中途停下 → 回 ① 等意图态(不掉进空屏黑洞)。

### 状态 ③ 策展 —— 唯一的灵魂屏:看懂、说句话就改、存

草稿成形。顶部 `收自:X` 仍在(猜错就在这条上改方向);中间是用户唯一要审的**正文**;
若起草时抹过 secret,加一行灰字让用户**看见 AI 替他把住了安全**;底部一句话点破机制
"我来改"——让新手第一次就敢相信"打字是在指挥 AI,不是在破坏草稿"。

```
╭ 新建技能 · 草稿 ────────────────────────────────────────────╮
│                                                             │
│   收自：配置生产环境的部署流程        （说一句可改方向）      │
│                                                             │
╰─────────────────────────────────────────────────────────────╯

  名称       部署生产流程
  什么时候用  要把服务部署到生产环境时

  ── 正文 ──────────────────────────────────────────────────

  1. 先 pnpm build 全量构建，确认无 error
  2. 镜像推 registry，tag 用 commit sha
  3. 滚动发布，盯住健康检查

  · 已自动抹掉对话里的 1 处密钥，不会写进技能
  · 归到工作场景；需要的话说一句改成日常也行
  ──────────────────────────────────────────────────────────
  这是草稿。在下面用大白话说要改哪，我来改；满意了 Ctrl+S 存下来。

 ╭─────────────────────────────────────────────────────────────╮
 │ 比如：描述再具体点 / 补一条回滚的坑                           │
 ╰─────────────────────────────────────────────────────────────╯
  放弃 Esc            存下来 Ctrl+S    让我改 Enter    自己改 Ctrl+E
```

> · 顶部 C 位只留身份 + `收自:X`。**删了**"落点 own/…"(用户看不懂的内部目录、保存细节);
> "模式"不占顶栏,降为正文末尾一行白话灰字、并明示可改。C 位全给正文——它才是要审的。
> · `名称` 的值加 **bold**(值亮、标签 dim),一眼锁定技能叫什么;字段值按显示宽度对齐成
> 一条隐形竖线。
> · 教学拧成一句、统一主语"**我来改**":正文下方那句 + hint 的 `让我改 Enter` 口径一致,
> 消除"Enter 是不是进手动编辑"的误解。`存下来 Ctrl+S` 提到 hint 前列——满意点头就存
> 才是默认终点。
> · 脱敏行 / 模式行**条件出现**:没抹到 secret 就不显示脱敏行,不无中生有。

### 状态 ④ 落地 —— 让"画面消失"翻译成"成功落地",且只承诺 v1 真有的

按 Ctrl+S 后,编辑屏的 alt-screen 会整屏退出、那份草稿瞬间消失。若没承接,这一下像"丢了"。
所以**退屏前先就地闪一帧成功**,退屏后再用醒目三行在对话流接住用户。

退屏前(编辑屏内,约 300–500ms):

```
  ◆ 存好了
```

退屏后(回到主对话流):

```
  ✓ 技能「部署生产流程」存好了。

  · 回到聊天输入框，打 /部署生产流程 就能唤起它
  · 下次你聊到往生产环境发东西，我也会检索到它
```

> · **去掉了"在相关场景里我也会自动想起它"**——那是 v2 技能管家(主动唤醒)的能力,v1
> 做不到,承诺了就是失信、会被用户当场证伪。换成 v1 真有的两条:手动 `/<name>` 唤起 +
> description 驱动的检索命中(用"你聊到往生产环境发东西"这种**具体场景**,代替抽象的
> "相关场景",既诚实又不让人担心乱触发)。

## 五、边角态(同样不让用户懵)

**放弃(Esc)—— 最高频也最危险,给护栏。** 改了几轮的满意草稿不能一键蒸发;且输入框里
Esc 的肌肉记忆是"清空输入",这里却是"毁掉整个技能",按"先小后大"分层:

```
  输入框非空时   ：第一次 Esc 先清空输入（符合终端习惯）
  输入框为空、有草稿：Esc 落一行二次确认 ↓
```

```
  按 Esc 放弃这份草稿（改了 3 轮，存了就没了）。再按一次确认，或 Ctrl+S 存下来。
```

> 什么都还没起草(空草稿)时 Esc 可直接退、无需确认。

**起草失败 —— 不甩锅、不吓人、不丢意图。** 失败多是模型偶发没吐合法 JSON,与用户"说法"
无关。默认动作 = 用同一意图直接重试,把刚才的意图**预填回输入框**:

```
  ⚠ 这次没起草出来（偶尔会抽风），照原来的意思再说一遍就行。

 ╭─────────────────────────────────────────────────────────────╮
 │ 配置生产环境的部署流程                                        │   ← 预填回意图
 ╰─────────────────────────────────────────────────────────────╯
  放弃 Esc                                           重试 Enter
```

> 删掉"模型没给出有效草稿"这种开发者黑话——用户会误以为是自己说错了。

**自己写(Ctrl+E)—— 手写逃生口,任何时候可走。** 已有草稿则交接当前草稿;还没草稿(从 ①
直接按)则落一个空骨架文件交编辑器,跳过 AI。改完保存、回来按任意键读回(mtime 比对):

```
  ◇ 已用你的编辑器打开草稿。改完保存，回来按任意键读回。
```

## 六、评审采纳的关键改动(可追溯)

- **结构性反转**:删 B 态概括确认闸门 + proposing phase;"起草中"由独立屏改为草稿区
  就地显影。决策点收敛到"存 / 说句话改"。
- **四处硬伤**:① 保存屏去掉做不到的"自动想起它"(失信)→ 诚实版闭环;② 首次起草取消
  落回"等意图"态、不进空屏黑洞;③ Esc 放弃加二次确认 + 输入框 Esc 分层去二义;④ 草稿屏
  顶部删 own / mode 降级,C 位让给正文。
- **高杠杆加分**:脱敏可见(真抹了 secret 才显示一行灰字),一句话同时建信任 + 教育。
- **打磨**:教学统一"我来改"主语、"存"提为主操作、名称值 bold + 字段对齐、▸ 不再当
  装饰符(它是列表光标保留义,陈述句用「」承载)、停下全程只暴露 Esc 一个键。

## 七、落地影响(实现侧)

- **少一次 LLM、少一个 phase**:删 proposing 态与概括调用;`editor-controller` 的阶段机
  回到 `editing / drafting / external` 三态,drafting 渲染改为"骨架 + 就地显影"。
- **草稿屏顶部 `收自:X`**:取首句意图,或起草 prompt 里 AI 判定的主题字段(零额外调用)。
- **取消落点**:`cancelDraft` 区分有/无草稿——无草稿落回"等意图"输入态。
- **Esc 二次确认 + 输入框分层**:在 `handleEditorKey` 加一个 pending-discard 子态。
- **脱敏可见**:`draftSkill` 已过 `scrubSecrets`,让它回传"抹了几处"计数,草稿屏据此显隐灰字。
- **保存承接**:落盘成功 → 编辑屏内闪一帧 `◆ 存好了` 再退屏;退屏后三行诚实闭环。
- **动效定时重画**:drafting 期间 250ms timer tick,复用 status-bar ticker 范式。
- **必须 `pnpm cli` 自跑视觉**:80 / 120 / 40 列三档各跑一帧,确认 chrome 副标题走
  `chromeContentWidth + wrapToWidth` 折行(别截成半截)、字段值对齐、印鉴不串位、hint
  两端对齐不溢出。
