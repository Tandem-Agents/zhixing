# 需求思考工作台

> **状态**：当前无活跃需求思考。
>
> 本文件用于保留一个尚未完全收敛的需求想法，包括原始表达、阶段性理解、约束、证据、风险和迁出结论；需求收敛后迁入待做需求池、独立 draft 或 spec。

## 维护原则

1. **回到需求本质**：先说清用户真实需要什么，再谈设计形态。
2. **避免架构反推需求**：不能因为已有底座，就反过来为它寻找产品理由。
3. **复杂性内收**：真正好的能力应让用户更容易完成生活和工作，而不是学习新概念。
4. **及时迁出**：需求收敛后迁入待做需求池、独立 draft 或 spec，本文件不长期堆历史。

## 使用方式

1. 本文件一次只承载一个活跃需求思考。
2. 新想法进入时，先保留原始表达，再逐步整理为结构化内容。
3. 需求收敛后，将稳定结论迁入对应正式文档或需求池。
4. 迁出完成后，只清空具体需求内容，保留本工作台结构。
