# 知行 · 原创设计沉淀（Innovations）

> 收纳"独立思考得出的、具有独特产品价值"的设计成果。
>
> 这里**不是**：
> - 模块级规范（看 `research/design/specifications/`）
> - 工作台（看 `research/design/active-problem.md`）
> - 新功能草稿（看 `research/design/drafts-roadmap.md`）
> - 已沉淀的问题对齐（看 `research/design/problems/`）
> - 对外认知研究（看 `research/insights/` —— 那是学他家的）
>
> 这里**是**：经过深入思考、结合自身需求与场景推导出的设计方案。是"我们独立想清楚的东西"，与 `insights/`（"我们对外学到的东西"）形成对照。

## 收录原则

每篇文档应满足：

1. **经过深入思考**：不是常规方案的小调整，而是结合需求与场景认真推导的设计
2. **第一性**：从问题本质 / 用户真实需求出发，不是堆砌功能
3. **价值清晰**：能说清现有方案的局限 + 我们方案的取舍
4. **可落地**：不只是哲学，要落到机制 + 规则 + 实施路径

不满足以上要求 → 进 `specifications/` 或工作台。

## 文档结构

每篇遵循一致结构：

```
一、问题背景       —— 触发场景 + 实测数据
二、第一性分析     —— 跳出已有框架的本质拆解
三、已知方案对比   —— 行业现状 + 各自局限
四、设计核心       —— 我们的机制 + 规则
五、设计哲学       —— 一句话提炼的灵魂
六、价值           —— 与三方对比 + 颠覆点
七、关键参数       —— 已敲定值（不开放配置）
八、实施路径       —— 落地点（不展开，留 spec 链接）
```
