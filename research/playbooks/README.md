# Agent 行动指引 (Agent Playbooks)

> 给 agent 看的行动指引——在特定场景下「该怎么干」的可执行规程，不是设计、不是需求。

与 `design/` 的区别:`design/` 描述「系统该是什么」(需求 / 架构 / spec);`playbooks/` 描述「agent 干活时该怎么行动」。每个文件聚焦一类行动场景,沉淀可复用的执行纪律与避坑原则,供 agent 进场前对齐。

## 目录约定

- 本目录根级放**可提交、可复用**的通用行动指引（执行纪律 / 避坑原则）。
- `local/` 放 agent 干活时的**过程活文档**（如审核执行追踪），随工作实时变动、不沉淀为通用规程，**不提交**——已在根 `.gitignore` 忽略。
