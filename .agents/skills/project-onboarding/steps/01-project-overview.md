# 项目整体了解

## Objective

先对当前项目建立广度优先的基础认识，为下一步阅读核心架构文档准备上下文。

## Resources

None

## Instructions

1. 阅读 `AGENTS.md`，理解项目级指令、项目背景和工作约束。
2. 查看根目录项目元数据、工作区结构和主要入口文件。优先使用 `rg --files`、`Get-ChildItem` 等聚焦命令，再按需阅读 README、package/workspace 配置或其他根级说明文件。
3. 梳理主要包、运行入口、研究设计目录和测试/构建相关信息。此阶段只做整体了解，不深入分析单个模块实现。
4. 产出一份简洁的项目地图，说明项目是什么、主要目录负责什么、后续阅读架构文档时应带着哪些背景。

## Output

一份项目整体了解摘要。

## Completion Criteria

- 已总结项目目标和定位。
- 已识别主要包或目录的职责。
- 已记录从根级文件中发现的重要约定、命令或开发事实。
- handoff 足以支撑下一步阅读指定核心文档。

## Handoff

向下一步传递：

- 项目目标的一到两句话概括。
- 主要包或目录地图。
- 已发现的根级约定、命令或开发事实。
- 会影响核心文档阅读的不确定点。
