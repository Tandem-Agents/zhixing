# 多 Agent 协作看板 (agent-board)

> 单一工作目录下多个 agent 并发开发时的共享活看板。一个文件、人机同读。
> 进场的 agent：先读「原则区」→ 在「实例区」登记自己 → 在「工作区」开自己的块并维护。
> 任务由用户分派;本看板只负责让并发的 agent 彼此「看得见」,不替用户调度。

---

## 一、原则区（进场到收尾的开机自检）

按顺序照做:

1. **先读后动** —— 动手前读完本区,再扫一遍「实例区」和「工作区」,看清别的 agent 在跑什么、在改哪,避免撞车。
2. **登记自己** —— 到「实例区」末尾**追加你自己一行**(不预留空位、不去抢或改别人的行),状态置 `active`。
3. **守边界、只写自己的块** —— 在「工作区」开一个属于你的块,**只写它、绝不碰别人的块**。旁逸发现的问题记下来、不顺手改 —— 顺手改会扩大冲突面、模糊边界。
4. **声明影响范围** —— 在你的块里写清你会动哪些包 / 文件。这是防撞的核心:用户已按互不重叠分派,你的声明让这件事可被同伴核对。别偷懒。
5. **收尾即释放** —— 任务做完,把实例区状态改成 `done`;卡住了改成 `blocked` 并写明卡在哪,别让同伴干等。
6. **commit 归用户、错开 build** —— 绝不自主 `git commit` / `push`(多 agent 共享同一个 git index,统一由用户控制提交时机);构建前看一眼有没有人正在 build,别同时写 `dist/`、`node_modules`。

> 「实例区」「工作区」是**运行态**、时时在变,随手改即可。它的价值在运行时被各 agent 实时读写;这些变更要不要 commit 进 git,由用户统一决定。
